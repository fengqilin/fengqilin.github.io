docker 扩容

By feng

发表于 2017-10-30

某日，在kubernetes准备创建一个应用，起不来，查日志

大意是空间不够了，网上查查资料， docker默认为10G, 扩容，记录如下
环境 centos 7 lvm
查看信息如下：

已无可用空间

查看卷组信息

系统中存在一下叫centos的卷。

docker 的Lv name docker-pool ,也是存在于卷 centos，这是我扩容后的空间。

所以我们第一步加入新分区到 centos 卷中，然后给 docker-pool 分配空间

1，添加一块新盘。
2，添加后，在我的秕统里出现 /dev/sdc
用fdisk格式化 主分区，扩展分区都可以
格式化后，系统出现 /dev/sdc1
3 创建物理卷
pvcreate /dev/sdc1
4，扩展 centos 卷
vgextend centos /dev/sdc1
5,为docker-pool增加空间
lvextend -L +40G /dev/centos/docker-pool
6，正常扩展分区是
resize2fs /dev/centos/docker-pool
但是会报错，直拉重启docker 或电脑即可。

7， 查看 lvdisplay 是否增加
8，又可以玩 kubernetes 了

kube-proxy iptable转发问题

By feng

发表于 2017-09-29

接前面，建好一个master ,三个节点，发布了一个副本为2的nginx服务，
发现运行在其中两台机器上。按理在三个节点上都可以用serviceIP:port进行访问。
但实际情况是在没有运行的那台机器上是访问不了的，
在运行的机器上访问，随机会出现连接超时，进入一个docker修改文件后，发现访问转到本机时都很快，只要不是本机就会出现连接超时。
为了这个问题，还重建了一个集群，但问题一样，各种网上查找，总算找到答案
http://blog.opsnull.com/post/golang/%E5%AE%89%E8%A3%85k8s/
http://blog.clanzx.net/2013/08/22/rp_filter.html

原因参照资料说明；
解决方案:
A 各节点 sysctl -w net.ipv4.conf.all.rp_filter=2;
这个重启会失效，写入syscty文件最好

B 或者为kube-proxy指定 --masquerade-all=true 参数，

C 或者为kube-proxy指定 --cluster-cidr 参数
特此记录一下。好大一个坑，网上教程都说就可以访问了，也不知道究竟有没有试过。

接前面，准备节点直接使用 etcd集群所在机器

vm-h3 10.10.1.233
后期增加
10.10.1.234 vm-h4
10.10.1.235 vm-h5
节点需要启动：
Node上面运行三个模块：kubelet、kube-proxy、runtime(docker)。
Node自己向服务器注册，所以服务器是不用知道节点的，只需要节点做好配置文件，启动对应的服务程序即可。
以下 操作在 vm-h1 10.10.1.231 这台机器。
A 安装 yum install kubernetes docker也会动装上。
同样/etc/kubernetes/config 是一些公有配置参数

KUBE_MASTER= 需要指定到master机器及端口

启动文件：
/usr/lib/systemd/system/kubelet.service
/usr/lib/systemd/system/kube-proxy.service
/usr/lib/systemd/system/docker.service

B docker先运行起来
systemctl start docker
systemctl enable docker 随机启动
C 编辑配置文件 vi /etc/kubernetes/kubelet

##
# kubernetes kubelet (minion) config

# The address for the info server to serve on (set to 0.0.0.0 or "" for all interfaces)
KUBELET_ADDRESS="--address=0.0.0.0"

# The port for the info server to serve on
KUBELET_PORT="--port=10250"

# You may leave this blank to use the actual hostname
KUBELET_HOSTNAME="--hostname-override=vm-h1"

# location of the api-server 
KUBELET_API_SERVER="--api-servers=http://10.10.1.241:8080"

# pod infrastructure container
KUBELET_POD_INFRA_CONTAINER="--pod-infra-container-image=registry.access.redhat.com/rhel7/pod-infrastructure:latest"

# Add your own!
KUBELET_ARGS=""

KUBELET_HOSTNAME=”–hostname-override=vm-h1” 设置唯一名，这就以主机名配置的

D kube-proxy配置文件，无须特别配置 /etc/kubernetes/proxy，因为一些参数从通用配置文件里加载了
E systemctl restart docker kubelet kube-proxy
查看状态及到 master 机器执行 kubectl get node
NAME STATUS AGE
vm-h1 Ready 5d
同理，在其他机器执行同样的配置操作。
