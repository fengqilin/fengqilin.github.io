kubernetes发布docker私服

By feng

发表于 2017-09-29

文章目录

现在需要建一个docker私服，网上教程大部分都是直接运行docker私服镜像，
但我们即然弄kubernetes，就kubernetes来管理吧。
前期建了一个单节点的集群，就是master ,node都在本机的一个集群。
想着在上面发布一个docker私服，用于另一个集群的测试等工作，
当然也可以在另一个多节点的集群上发布，那就是需要把镜像存放目录用NFS或别的方式共享持载。除了镜像存储不同，别的都一样。
因为docker 私服访问需要https，原本计划参照网上启用http方案，便试了添加各种参数无效，正好手里有公司的一个泛域名的证书，当然也可能自签证书。
主机地址： 10.10.1.201
为避免一些麻烦，通过hosts或自建dns把 docker.bofide.cn
指向 10.10.1.201,在集域网内可以通过docker.bofide.cn进行访问。
公钥： bofide2year.cn ,私钥 private2year.key
参照文档：
https://github.com/kubernetes/kubernetes/tree/master/cluster/addons/registry/tls
步骤：
1 create secret
进入这两个文件所在目录，可以少输路径~~

我是放在 namespace=default，要放别的空间，就改空间名再接着建
查看一下。

2 编写docker-registry-rc.yaml 文件

kubectl create -f docker-registry-rc.yaml

查看有没有问题 ,如果有问题查看日志解决，一般情况下只要几个路径对得上就没问题

3 编写发布 service docker-registry-service.yaml

kubectl create -f docker-registry-service.yaml
网上有都程说还需要把
还需要将bofide2year.cn证书拷贝到集群每个node的
/etc/docker/certs.d/docker.bofide.cn/ 目录下。但我测试不需要，按道理也是不需要的。
